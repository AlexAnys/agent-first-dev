# 术语表

跨模块术语速查，涵盖 8 周课程中的关键概念。每个术语用日常类比解释，帮助非技术背景的读者快速理解。

---

## 基础概念

**提示工程 (Prompt Engineering)**
给 AI 写指令的技巧。就像你给一个特别聪明但非常较真的实习生布置任务 -- 你说得越清楚、给的例子越好，它完成得就越好。

**大语言模型 (LLM / Large Language Model)**
ChatGPT、Claude 等工具背后的核心技术。它读过海量文本，学会了理解和生成语言。你可以把它想象成一个读过整个图书馆的助手，虽然它不"理解"内容，但能根据模式给出相当有用的回答。

**少样本提示 (Few-shot Prompting)**
在提示中给 AI 几个示例，让它照着做。就像你教新员工的时候说"参考这几个已经做好的案例，照着这个格式来"。

**思维链 (Chain-of-Thought)**
要求 AI 一步一步推理，而不是直接给答案。就像数学考试要求"写出解题过程" -- 过程写出来了，答案往往更准确。

**自一致性 (Self-Consistency)**
让 AI 对同一个问题回答多次，然后选最常出现的答案。类似于"三个臭皮匠赛过诸葛亮" -- 多问几次取共识，比只问一次更可靠。

**检索增强生成 (RAG / Retrieval-Augmented Generation)**
先从你的文档库中检索相关资料，再让 AI 基于这些资料回答问题。就像一个员工在回答客户提问之前，先翻阅公司手册找到相关条款，然后再组织语言回复。

**反思 (Reflexion)**
让 AI 审视自己之前的输出，找出错误并改进。类似于写完文章后自己校稿一遍 -- 第二遍往往能发现第一遍的疏漏。

**工具调用 (Tool Calling / Function Calling)**
AI 不只是生成文字，还能调用外部工具完成任务，比如搜索网页、查询数据库、执行代码。就像你的助手不只会写报告，还会帮你查资料、订机票、发邮件。

## 开发工具与流程

**API (Application Programming Interface)**
应用程序编程接口。不同软件之间沟通的标准方式。就像餐厅的菜单 -- 你不需要知道厨房怎么做菜，只需要看菜单点单，厨房就知道该给你做什么。

**前端 / 后端 (Frontend / Backend)**
前端是用户看到和操作的界面（网页、App 的按钮和画面）；后端是服务器上处理数据和逻辑的部分。就像餐厅的前厅（顾客看到的）和后厨（顾客看不到但实际做菜的地方）。

**数据库 (Database)**
专门用来存储和管理数据的系统。就像一个超级有组织的电子档案柜，可以快速存入、查找、更新和删除信息。

**ORM (Object-Relational Mapping)**
一种让你用编程语言直接操作数据库的工具，不需要写专门的数据库查询语句。就像有了同声传译，你说中文，它自动帮你翻译成数据库能听懂的语言。

**全栈 (Full-Stack)**
既包含前端又包含后端的完整开发。一个"全栈"项目意味着从用户界面到服务器逻辑再到数据库，全部自己搭建。

**代码编辑器 (Code Editor / IDE)**
写代码的专用工具，比普通记事本多了语法高亮、自动补全、错误提示等功能。常用的有 VS Code、Cursor 等。就像 Word 之于写文章，代码编辑器之于写代码。

**版本控制 (Version Control / Git)**
追踪代码每一次修改的系统。就像文档的"修订历史"功能，你可以看到谁在什么时候改了什么，还能随时回退到之前的版本。Git 是目前最流行的版本控制工具。

**分支 (Branch)**
从主代码中"分出来"的一个独立副本，可以在上面放心修改而不影响主代码。就像在正式文件的基础上复印一份，在复印件上标注修改意见，确认没问题后再合并回正式文件。

**拉取请求 (Pull Request / PR)**
请求将你在分支上的修改合并到主代码中。就像你把修改好的复印件交给主管审批，主管同意后才会把修改合并到正式文件里。

**代码审查 (Code Review)**
其他人检查你写的代码，提出意见和改进建议。就像论文的同行评审 -- 多一双眼睛看，能发现自己看不到的问题。

**持续集成 (CI / Continuous Integration)**
每次代码提交后，自动运行测试和检查，确保新代码没有破坏已有功能。就像工厂流水线上的质检环节 -- 每个零件装上去之后都自动检测一遍，发现问题立刻报警。

## AI 辅助开发

**Agentic Coding**
让 AI 代理主动地帮你编写代码。不是你写一行问一下，而是你描述目标，AI 自主规划步骤、写代码、调试、甚至自己发现和修复问题。你更像是项目经理，AI 是动手干活的工程师。

**Vibe Coding**
一种更随意的 AI 辅助编码风格 -- 你大致描述想要什么，AI 来生成代码，你看看结果对不对，不对就继续调整。强调快速迭代和直觉驱动，适合原型设计和探索阶段。

**MCP (Model Context Protocol)**
模型上下文协议。一种让 AI 工具连接外部数据源和服务的标准化方式。就像 USB 接口 -- 有了统一标准，各种设备（数据源）都能方便地接入 AI。

**CLAUDE.md**
放在项目根目录的一个特殊文件，用来告诉 Claude（AI 助手）这个项目的背景信息、编码规范和注意事项。就像你给新入职的同事准备的"项目须知"文档。

**斜杠命令 (Slash Commands)**
在 AI 编码工具中以 `/` 开头的快捷指令，比如 `/commit`、`/review`。就像聊天软件里的快捷操作 -- 输入一个简短命令就能触发一系列自动化动作。

**子代理 (SubAgents)**
主 AI 代理为了完成复杂任务而派出的"小助手"。就像你把一个大项目拆成几个子任务，分别交给不同的人同时去做，最后汇总结果。

**代码扫描 (Code Scanning / SAST)**
自动检查代码中是否存在安全漏洞或常见错误的工具。SAST 是 Static Application Security Testing（静态应用安全测试）的缩写。就像给代码做"体检"，在上线之前发现潜在隐患。

**AI 代码审查 (AI Code Review)**
用 AI 自动审查代码质量、风格和潜在问题。就像请了一个不知疲倦的审稿人，每次你提交代码都会帮你通读一遍并给出改进意见。
